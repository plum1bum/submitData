# Проект на Python и Django для работы с данными перевалов

Этот проект разработан на Python с использованием Django и предоставляет REST API для управления данными перевалов. Мы реализовали методы, позволяющие добавлять и извлекать информацию о перевалах. В этом README подробно рассматриваются задачи, выполненные в рамках разработки, а также документация к REST API.

## Описание проекта

Проект предназначен для сбора и управления данными о перевалах. Пользователи могут отправлять информацию о новых перевалах через REST API, что позволяет хранить эти данные в базе данных и предоставлять доступ к ним для последующей обработки и анализа.

## Основные задачи

Проект разбит на три ключевых задачи:

### Создание базы данных

Создана база данных, использующая SQLite для хранения информации о перевалах. Модель Record описывает структуру данных, включая поля, такие как название перевала, координаты и т.д.

### Создание класса для работы с данными

Разработан класс, позволяющий взаимодействовать с базой данных. Этот класс предоставляет методы для добавления записей и гарантирует валидацию входящих данных.

### Написание REST API

Создан REST API, который позволяет пользователям взаимодействовать с данными перевалов через HTTP-запросы. Реализован метод POST submitData, который принимает JSON-объект с информацией о перевале и добавляет запись в базу данных.

## Документация к REST API

### Основные эндпоинты
```
- POST /submitData — добавление нового перевала.
- GET /submitData/<id> — получение информации о перевале по ID.
```
### POST /submitData

#### Описание

Метод обрабатывает JSON-объект, содержащий информацию о новом перевале. После успешного добавления возвращает JSON с данными о созданном объекте.

#### Запрос
```
POST /submitData HTTP/1.1
Host: your-api-host.com
Content-Type: application/json
```
```
{
  "beauty_title": "Перевал Ушко",
  "title": "Ушко",
  "other_titles": "Перевал 1, Перевал 2",
  "connect": "Хороший маршрут",
  "coords": {
    "latitude": 44.1234,
    "longitude": 42.6789,
    "height": 2000
  },
  "level": {
    "winter": "Высокий",
    "summer": "Средний",
    "autumn": "Низкий",
    "spring": "Средний"
  },
  "user": {
    "email": "user@example.com",
    "fam": "Иванов",
    "name": "Иван",
    "otc": "Иванович",
    "phone": "+79000000000"
  }
}
```

#### Ответ

Если запрос успешен, сервер вернет статус 201 и JSON-объект с данными о созданном перевале:
```
{
  "id": 1,
  "beauty_title": "Перевал Ушко",
  "title": "Ушко",
  "other_titles": "Перевал 1, Перевал 2",
  ...
}
```

### GET /submitData/<id>

#### Описание

Метод возвращает информацию о перевале по заданному ID.

#### Запрос
```
GET /submitData/1 HTTP/1.1
Host: your-api-host.com
```

#### Ответ

Если объект найден, сервер возвращает статус 200 и данные о перевале:
```
{
  "id": 1,
  "beauty_title": "Перевал Ушко",
  ...
}
```

Если объект не найден, возвращается статус 404.

## Примеры вызова REST API с хостинга

### Пример добавления нового перевала
```
curl -X POST https://your-api-host.com/submitData \
-H "Content-Type: application/json" \
-d '{
  "beauty_title": "Перевал Ушко",
  ...
}'
```

### Пример получения перевала по ID
```
curl -X GET https://your-api-host.com/submitData/1
```
